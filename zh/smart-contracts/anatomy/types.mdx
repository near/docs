让我们讨论智能合约使用哪些类型来输入和输出数据，以及这些数据在合约代码中如何存储和处理。

<ExplainCode languages="rust,js,python">

<Block highlights='{"js":"5,8,13"}' fname="hello-near">

    ### 原生类型
    智能合约可以使用 JS 原生类型接收、存储和返回数据：
    - `string`
    - `number`
    - `boolean`
    - `Array`
    - `Map`
    - `Object`
    - `BigInt`

</Block>

<Block highlights='{"rust":"6,13,22,27"}' fname="hello-near">

    ### 原生类型
    智能合约可以使用以下 Rust 类型接收、存储和返回数据：
    - `string`
    - `i8-i32/u8-u32`
    - **`u64/128`**：建议使用 SDK 类型 `U64` 和 `U128`
    - `bool`
    - `HashMap`
    - `Vector`

</Block>

<Block highlights='{"python":"11,16"}' fname="hello-near">

    ### 原生类型
    智能合约可以使用以下 Python 类型接收、存储和返回数据：
    - `str`
    - `int`
    - `float`
    - `bool`
    - `list`
    - `dict`
    - `set`
    - `bytes`
    - `None`

</Block>

<Block highlights='{"rust": "1,15,24,81"}' fname="auction" type='info'>

<Warning>
**`U64/U128`**

智能合约可以存储 `u64` 和 `u128`，但这些类型需要转换为 `string` 用于输入/输出，因为 JSON 无法序列化超过 52 位的值。

为了简化开发，SDK 提供了 `U64` 和 `U128` 类型，存储时自动转换为 `u64/u128`，用作输入/输出时转换为 `string`。

</Warning>

</Block>

<Block highlights='{"python": "26,28"}' fname="auction" type='info'>

<Tip>
**Python 大数字**

Python 的 `int` 类型具有无限精度，因此它可以处理大整数（如 yoctoNEAR 值）而无需任何特殊处理。所有值都会自动正确地序列化和反序列化。

</Tip>

</Block>

<Block highlights='{"js":"3-6", "rust": "6-9", "python": "10-13"}' fname="auction">

    ### 复杂对象
    智能合约可以存储和返回复杂对象

    **注意：** 对象始终以 JSON 格式接收和返回

</Block>

<Block highlights='{"rust": "4"}' fname="auction">

    #### 序列化器
    将用作输入或输出的对象需要序列化为 JSON，添加 `#[near(serializer=json)]` 宏

    将存储在合约状态中的对象需要序列化为 Borsh，添加 `#[near(serializer=borsh)]` 宏

</Block>

<Block highlights='{"python": "7,14,17,20"}' fname="auction">

    #### 序列化
    Python 对象使用 JSON 自动序列化和反序列化用于输入/输出，使用 Pickle 用于内部存储。

    列表和字典等复杂嵌套对象无需额外配置即可直接使用。

</Block>

<Block highlights='{"js": "5,10,28"}' fname="auction">

    ### 处理代币
    `$NEAR` 代币在 JS 中类型化为 `BigInt`，其值以 `yoctonear` 表示

    **注意：** 1 NEAR = 10^24 yoctoNEAR

</Block>

<Block highlights='{"rust": "8,28,45"}' fname="auction">

    ### 处理代币
    `$NEAR` 代币通过 `NearToken` 结构体处理，该结构体提供以 `yoctonear`、`milinear` 和 `near` 表示值的方法

    **注意：** 1 NEAR = 10^24 yoctonear

</Block>

<Block highlights='{"python": ""}' fname="auction">

    ### 处理代币
    在 Python 中，`$NEAR` 代币表示为整数，值以 `yoctoNEAR` 为单位。
    `near_sdk_py.constants` 模块提供了 `ONE_NEAR` 和 `ONE_TGAS` 常量。

    **注意：** 1 NEAR = 10^24 yoctoNEAR

</Block>

<Block highlights='{"js": "4,29", "rust": "7,46", "python": ""}' fname="auction">

    ### 账户
    SDK 提供了一种特殊类型来处理 NEAR 账户，它会自动检查账户地址是否有效

</Block>

<Block highlights='{"python": ""}' fname="auction">

    ### 账户 ID
    在 Python 中，NEAR 账户 ID 表示为 `str` 类型。当账户 ID 用于合约调用或跨合约交互时，SDK 会执行验证。

</Block>
