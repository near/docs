Welcome! In this guide we will help you navigate NEAR tech stack, so you can build Web3 applications from start to finish in no-time.

We'll start from a simple auction contract and slowly build on top of it to create a full Web3 application to carry out on-chain auctions.

By the time you finish this tutorial, you will have learned several concepts and how to use many key primitives along the way:

- [Creating a simple smart contract](./1.1-basic#the-contracts-state)
- [Writing tests for a contract](./1.2-testing)
- [Deploying a contract to testnet](./1.3-deploy)
- [Locking a contract](./1.3-deploy#locking-the-contract)
- [Creating a frontend to interact with the contract](./2.1-frontend)
- [Using an indexing API to view historical bids](./2.2-indexing)
- [Making cross-contract calls](./3.1-nft#transferring-the-nft-to-the-winner)
- [Using Non-Fungible Tokens](./3.1-nft)
- [Using Fungible Tokens](./3.2-ft) 
- [Modifying a factory contract to deploy your own contracts](./4-factory)

## Prerequisites

Before starting, make sure to set up your development environment! 

<Accordion title="Working on Windows?">

  See our blog post getting started on NEAR using Windows for a step-by-step guide on how to setup WSL and your environment

</Accordion>

<Tabs>
  <Tab title="🌐 JavaScript">

    ```bash
    # Install Node.js using nvm (more option in: https://nodejs.org/en/download)
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
    nvm install latest

    # Install the NEAR CLI to deploy and interact with the contract
    curl --proto '=https' --tlsv1.2 -LsSf https://github.com/near/near-cli-rs/releases/latest/download/near-cli-rs-installer.sh | sh
    ```

  </Tab>

  <Tab title="🦀 Rust">

    ```bash
    # Install Rust: https://www.rust-lang.org/tools/install
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

    # Contracts will be compiled to wasm, so we need to add the wasm target
    rustup target add wasm32-unknown-unknown

    # Install the NEAR CLI to deploy and interact with the contract
    curl --proto '=https' --tlsv1.2 -LsSf https://github.com/near/near-cli-rs/releases/latest/download/near-cli-rs-installer.sh | sh

    # Install cargo near to help building the contract
    curl --proto '=https' --tlsv1.2 -LsSf https://github.com/near/cargo-near/releases/latest/download/cargo-near-installer.sh | sh
    ```

  </Tab>

</Tabs>

We will be using [NEAR CLI](/tools/cli) to interact with the blockchain through the terminal, and you can choose between JavaScript and Rust to write the contract.

## Overview

This series will touch on different level of the NEAR tech stack. Each section will be independent of the previous one, so feel free to jump into the section that interests you the most.

#### 1. Smart contracts 101
1. [The Auction Contract](./1.1-basic): We cover a simple auction smart contract                      
2. [Testing the Contract](./1.2-testing): Learn how to test your contract in a realistic environment
3. [Deploying the Contract](./1.3-deploy): Deploy your contract to the NEAR blockchain

#### 2. Frontends 101

1. [Creating the frontend](./2.1-frontend): Lets learn how to connect a frontend with your smart contract
2. [indexing historical data](./2.2-indexing): Use APIs to keep track of historical bids

#### 3. Using Primitives
1. [Giving an NFT to the Winner](./3.1-nft): Give the highest bidder an NFT to signal their win            
2. [Integrating Fungible Tokens](./3.2-ft): Allow people to use fungible tokens to bid (e.g. stable coins)
3. [Updating the frontend](./3.3-new-frontend): Update the frontend to use the extended functionality of the contract.

#### 3. Auction Factory
1. [Creating a factory](./4-factory): Allow users to easily deploy and initialize their own auction contracts
